Phishing Email Detector with Machine Learning

Category: Project

Introduction

The project aims to design and implement a Machine Learning (ML) model to detect abnormal emails, whether due to phishing, spamming, etc. The problem is framed as a binary classification issue.
This project selects four models, logistic regression, SVM, plain Bayes and random forest, and combines three feature extraction methods, TF-IDF, bag-of-words and embedding, to train an irregular mail monitoring model through model conditioning. The best model AUC reached 0.9970.

Contribution

  • Completion of the data cleaning portion of the program to remove duplicates/blank data.
  • Completion of data preprocessing procedures for removal of redundant characters, stop words, lexical restoration, etc.
  • Used bag-of-words to count word frequencies on the data, saved the results, and ploted the word cloud.
  • Calculated the percentage of gibberish words to help understand the data.
  • Dealt with the existing processed normal and abnormal data, labeled, merged and split them into a training dataset and a validation dataset.
  • Performed feature extraction using TFIDF and feature selection using both chi-square and Feature-Label Correlation Analysis.
  • Re-generated the data preprocessing results, performed feature extraction according to the original data ratio and the 10:1 data ratio respectively, and compared the results.
  • Trained the logistic regression model, test the model performance.
